To create a new host:

sysync --newhost=waffles
sysync --edithost=waffles

You'll see:
#users:
#   - uid: 0
#     username: root
#     homedir: /root
#     shell: /bin/bash
#     password: '$6$928b679b70731fc7$OjB.vI0hI4PWC9ObsudW3ITZMBjo7Rfs6Dd5vQ80XZM0A6NU6EQqIVQAI3T90T5Bz3K9Vfha0cp176IAHaNQQ.'
#     ssh_keys:
#        - here
#files:
#   - remote_path: '/etc/foo'
#     file: 'defaults/foo'
#     chmod: 777
#     owner: root
#     group: root
user_groups:

If you uncomment the root user, in users, you can override the password and ssh keys from the default.
The other users in default, not seen here, will still be imported to the system.

The `user_groups` setting indicates which users should be imported to the system.

If we have a group called sysadmin and foodancers and we only want users from these two groups, we'd want this:

user_groups:
   - sysadmin
   - foodancers

The groupname 'all' will import all user accounts in sysync.


Hosts are mapped to actual machines with the sysync --edithosts commend.

You'll see this:

hosts:
   hostnamehere:
      - server.address.here

Let's say you want to deploy the waffles host instance two two machines, waffle1.ohpasw.com and waffle2.ohpasw.com:

You'd want to do this:

hosts:
   waffle:
      - waffle1.ohpasw.com
      - waffle2.ohpasw.com
